## 思考過程

我自己的思考步驟如下

1. 用 reduce 算出總乘積，透過 map 把自己除掉 - 碰到 0 會掛所以不可行
2. 雙 loop 暴力解
3. 從暴力解導出 1 pass 的解法
4. 針對 Swift 的弱點優化

### ① 算出總成績再分別把自己除掉

最直覺蹦出這個想法，程式碼會如下：

``` swift
var product = nums.reduce(1, *)
return nums.map { product / $0 }
```

不過當陣列裡面有 0 的話就會掛掉。

### ② 雙 loop 暴力解

於是就先來塵歸塵土歸土，先從暴力解把基本解法寫出來，再來從裡面找線索。

``` swift
var result = Array(repeating: 1, count: nums.count)
for outer in 0..<nums.count {
    for inner in 0..<nums.count {
        if outer != inner {
            result[outer] *= nums[inner]
        }
    }
}
return result
```

問題是超大的陣列就會超時。

### ③ 分析暴力解找解法

以陣列 [2, 4, 3, 6] 為例 （未選這個是因為他不是排序好的數列）

暴力解的步驟可以看成這樣：

``` text
 nums [2, 4, 3, 6]  [2, 4, 3, 6]       [2, 4, 3, 6]
      ------------  ------------       ------------
       -             -  2               -  2  2  2
       4             4  -         ...   4  -  4  4
       3             3  3               3  3  -  3
       6             6  6               6  6  6  -
```

縱向的進展是 inner loop ，橫向的進展是 outer loop 。

``` text
nums [2, 4, 3, 6]
     ------------
      -  2  2  2
      4  -  4  4
      3  3  -  3
      6  6  6  -
```

來觀察這個表主要可以觀察出以下幾點

- 可以分成兩個部分，右上和左下
- 右上和左下每個步驟都可以視為一連串的 memoization 動作
  - 右上： index 位置的值是 index - 1 乘積乘上 nums[index - 1]
    - 例： index `3` 位置的值是 index `2` 的乘積 (2) 再乘上 nums[`2`] 也就是 4 。因此在右上處理中，這個位置的值會是 2 * 4
  - 左下同理，只是是取 index + 1 在左下處理過程中的結果

基於觀察到的這幾兩點，就可以開始寫 code 了。完成後如下：

``` swift
class Solution {
    func productExceptSelf(_ nums: [Int]) -> [Int] {
        // Edge case
        if nums.count <= 1 { return nums }
        
        // 先把結果 array 建好
        var result = Array(repeating: 1, count: nums.count)
        
        // 右上部分
        for index in 1..<nums.count {
            result[index] = result[index - 1] * nums[index - 1]
        }
        
        // 左下部分
        var secondCurrent = 1
        for index in (0..<(nums.count - 1)).reversed() {
            secondCurrent *= nums[index + 1]
            // 在處理左下的同時，順便把右上處理的結果乘出來
            result[index] *= secondCurrent
        }
        
        return result
    }
}
```

這邊執行完之後，會發現效能很差 - beats 76.85%
但是這其實是 Swift 的問題 XDDDD
根據經驗 Leetcode 上的 Swift 在處理 for-in 的時候會拖慢 runtime ，因此在這邊可以再把它優化一下

### ④ 針對 Swift 的弱點優化

要先說，這個變化會變得有點不太像 Swift ，不過為了讓 runtime 快一點不仿也可以試試看。這一步主要就是把元兇 for-in 全部換成 while 。

``` swift
class Solution {
    func productExceptSelf(_ nums: [Int]) -> [Int] {
        if nums.count <= 1 { return nums }
        var result = Array(repeating: 1, count: nums.count)
        
        var firstIndex = 1
        while firstIndex < nums.count {
            result[firstIndex] = result[firstIndex - 1] * nums[firstIndex - 1]
            firstIndex += 1
        }
        
        var secondIndex = nums.count - 2 // came from (nums.count - 1) - 1
        var secondCurrent = 1
        while secondIndex >= 0 {
            secondCurrent *= nums[secondIndex + 1]
            result[secondIndex] *= secondCurrent
            secondIndex -= 1
        }
        
        return result
    }
}
```

做了這一個改變， runtime 就往前進到 beats 98.43% 了 XDDDD

## 複雜度分析

令 n 為 nums 的元素數量。

- 時間複雜度：O(n)
- 空間複雜度：除了 result 的 array 花了 O(n) 之外，就是 O(1)
